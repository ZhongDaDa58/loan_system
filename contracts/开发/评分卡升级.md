这是为你定制的**第四阶段：智能风控系统升级（7天冲刺计划）**。

这个阶段的目标是将你训练好的 Python 模型转化为 Java 后端的自动化决策能力，实现从“人工审核”到“智能秒批”的跨越。

### 🎯 第四阶段总览
*   **核心目标**：集成 `model_config.json`，实现基于逻辑回归评分卡的自动化审批。
*   **最终成果**：用户提交申请后，系统能在毫秒级内完成风险打分、自动判定（通过/拒绝/人工），并生成可解释的风控报告。

### 💡 关键节点说明

#### 1. 基础建设阶段 (Day 1)
*   **重点**：确保 JSON 里的 `score_mapping` 能被 Java 高效检索。
*   **成果**：你会拥有一个全局单例的 `ModelConfig` 对象，随时供 Service 调用。

#### 2. 核心算法阶段 (Day 2)
*   **重点**：处理边界值（如收入正好等于 3115 时属于哪个区间）。
*   **成果**：实现一个 `calculateScore(UserProfile)` 方法，返回整数分数。

#### 3. 业务串联阶段 (Day 3)
*   **重点**：与第三阶段的合同模块联动。
*   **成果**：**全自动闭环**。如果分数 > 650，系统自动生成合同草稿；如果 < 600，直接反馈拒绝理由。
