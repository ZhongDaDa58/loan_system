### 💡 关键技术难点预警

1.  **中文字体支持**：PDF 库默认不支持中文，你需要在 `resources` 下放入 `simsun.ttc`（宋体）文件，并在代码中配置字体路径。
2.  **图片定位**：在 HTML 转 PDF 时，签名图片的定位可能会跑偏。建议使用 CSS 的 `position: absolute` 配合 PDF 库的绝对定位功能进行微调。
3.  **文件大小控制**：如果签名图片太大，生成的 PDF 会很慢。建议在嵌入前对 Base64 图片进行压缩。

### 📋 电子合同模块接口清单

#### 1. 生成并签署合同接口
*   **作用**：当贷款审批通过后，调用此接口生成 PDF，嵌入用户签名和平台公章，并计算哈希值。
*   **地址**：`POST /api/v1/contract/generate`
*   **输入**：`{ "loanApplicationId": 1001 }`
*   **输出**：
    ```json
    {
      "code": 200,
      "msg": "合同生成成功",
      "data": {
        "contractNo": "LOAN-20260508-001",
        "fileUrl": "/contracts/2026/LOAN-20260508-001.pdf",
        "hashValue": "a1b2c3d4..." // SHA-256 指纹
      }
    }
    ```


#### 2. 获取我的合同列表接口
*   **作用**：用户在“个人中心-我的合同”页面查看历史签署记录。
*   **地址**：`GET /api/v1/contract/my-list`
*   **输入**：无（从 Token 获取 userId）
*   **输出**：
    ```json
    {
      "code": 200,
      "data": [
        {
          "id": 1,
          "contractNo": "LOAN-20260508-001",
          "loanAmount": 5000.00,
          "signTime": "2026-05-08 14:30:00",
          "status": "SIGNED" // SIGNED: 已签署, PENDING: 待签署
        }
      ]
    }
    ```


#### 3. 获取合同详情/下载接口
*   **作用**：用户点击某份合同后，在线预览或下载 PDF 文件。
*   **地址**：`GET /api/v1/contract/detail/{contractNo}`
*   **输入**：路径参数 `contractNo`
*   **输出**：返回 PDF 文件的二进制流（`application/pdf`），或者返回包含下载链接的 JSON。

#### 4. 合同存证验真接口 (模拟区块链)
*   **作用**：展示合同的法律效力，验证合同是否被篡改。
*   **地址**：`GET /api/v1/contract/evidence/{contractNo}`
*   **输出**：
    ```json
    {
      "code": 200,
      "data": {
        "contractNo": "LOAN-20260508-001",
        "hashValue": "a1b2c3d4...",
        "blockHeight": 102456, // 模拟区块高度
        "timestamp": "2026-05-08 14:30:05",
        "isValid": true // 校验结果：合同未被篡改
      }
    }
    ```

### 💡 关键逻辑说明

1.  **自动化触发**：通常 `generate` 接口不会由前端直接调用，而是由后端在**贷款审批通过**（`AuditController`）的逻辑中自动触发。
2.  **安全性**：`detail` 接口必须校验 `userId`，确保用户只能下载自己签署的合同。
3.  **存证模拟**：`evidence` 接口会读取数据库中存储的 Hash 值，并与当前 PDF 文件的 Hash 进行比对。如果一致，则返回 `isValid: true`。
